## What This Repo Is

A version-controlled job search and application automation system. It:
1. Stores Michael's master resume in Markdown
2. Provides a weekly job board dashboard (local browser UI) for reviewing and approving jobs
3. Auto-generates tailored resumes and cover letters via Claude API when a job is approved
4. Tracks all applications in a structured Markdown tracker
5. Supports auto-submission to job portals (in progress — see Pending Work)

## Weekly Workflow (How Michael Uses This)

1. `python tools/job_board.py` — opens dashboard at localhost:5000
2. Browse auto-fetched RemoteOK listings + paste in jobs from LinkedIn/Indeed manually
3. Click **👍 Approve** on interesting roles → auto-saves to tracker + JD file
4. Open a Claude Code session → ask Claude to generate tailored cover letter + resume
5. Claude saves outputs to `cover-letters/generated/` and `applications/resumes/`
6. Submit application; update tracker status to ✉️ Applied

## Key Technical Decisions Made

- **No API key dependency for core workflow** — Michael uses Claude Max via Claude Code sessions; Python tools are present but secondary
- **Flask for job board** — simple local server, no cloud needed, opens browser automatically
- **Tracker as source of truth** — `👍 Approved` status is the trigger for all automation
- **JDs saved to `applications/jds/<company_slug>_<date>.txt`** — naming convention used by apply.py to find JDs
- **Claude API model**: `claude-opus-4-7` with `thinking: {"type": "adaptive"}` — no `budget_tokens` (removed on Opus 4.7)
- **Prompt caching**: resume content cached via `cache_control: {"type": "ephemeral"}` on system message
