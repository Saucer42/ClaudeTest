## What This Repo Is

A version-controlled job search and application automation system. It:
1. Stores Michael's master resume in Markdown
2. Uses Claude Code sessions as the job board — search, review, and approve jobs in chat
3. Auto-generates tailored resumes and cover letters via Claude Code (no API key needed)
4. Tracks all applications in a structured Markdown tracker
5. Submits applications via GitHub Actions (headless Playwright) — no local setup required

**Nothing needs to run locally.** The entire workflow runs through Claude Code sessions and GitHub.

## Weekly Workflow — No Local Setup Required

### Step 1 — Find jobs (Claude Code session)
Open a Claude Code session and say:
> "Search for data manager / director jobs in Toronto this week"

Claude uses WebSearch to find postings from LinkedIn, Indeed, company sites, etc. and presents
them in the chat. Say "approve the Shopify one" and Claude will:
- Update `applications/tracker.md` with `👍 Approved` status
- Save the job description to `applications/jds/<slug>_<date>.txt`
- Commit and push

### Step 2 — Generate materials (same Claude Code session)
> "Generate a cover letter and tailored resume for Shopify — here's the JD: [paste]"

Claude writes both files and saves them to:
- `cover-letters/generated/<slug>_<timestamp>.md`
- `applications/resumes/<slug>_<timestamp>.md`

Then commits and pushes. No API key needed — this runs inside the Claude Code session itself.

### Step 3 — Submit via GitHub Actions
1. Go to **GitHub → Actions → Submit Job Application → Run workflow**
2. Enter the company name (e.g. `Shopify`)
3. Leave **Dry run** checked (default) → click **Run workflow**
4. When the run finishes, click it → **Artifacts** → download the screenshot
5. Review the filled form in the screenshot
6. Re-run with **Dry run unchecked** to actually submit
7. The Action commits the tracker update (`✉️ Applied`) automatically

### What gets automated in Step 3
- Finds the latest generated cover letter and tailored resume for that company
- Converts the resume Markdown to a clean PDF
- Opens the Greenhouse or Lever application form in a headless Chrome browser
- Fills: name, email, phone, LinkedIn, uploads resume PDF, pastes cover letter
- Takes a full-page screenshot before (and after) submitting
- Updates tracker.md and pushes the commit

## Key Technical Decisions

- **No local setup required** — Claude Code session is the UI; GitHub Actions runs the browser
- **Dry run by default** — Actions workflow defaults to screenshot-only; submit is a deliberate second step
- **Tracker as source of truth** — `👍 Approved` status in tracker.md is what the Action looks for
- **Materials as Markdown** — generated files stay in Markdown in the repo; gha_submit.py converts to PDF at submission time
- **JDs saved to `applications/jds/<slug>_<date>.txt`** — naming convention used by gha_submit.py
- **Claude API model**: `claude-opus-4-7` with `thinking: {"type": "adaptive"}`
- **Prompt caching**: resume content cached via `cache_control: {"type": "ephemeral"}`
